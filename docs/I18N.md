# Internationalization (i18n)

## Overview

The app uses `expo-localization` + `i18n-js` for internationalization. French is the default language, with infrastructure ready for adding more languages.

## Usage

### In Components

```typescript
import { useTranslation } from '../i18n';

function MyComponent() {
  const { t, locale, setLocale } = useTranslation();

  return (
    <>
      <Text>{t('profile.title')}</Text>
      <Text>{t('auth.resetEmailSent', { seconds: '60' })}</Text>
    </>
  );
}
```

### API

- `t(key, options?)` - Translate a key with optional interpolation
- `locale` - Current language code (e.g., 'fr')
- `setLocale(language)` - Change language (persisted automatically)

## File Structure

```
src/i18n/
├── index.ts              # Public exports
├── types.ts              # Language enum & types
├── i18n.ts               # i18n configuration
├── useTranslation.ts     # React hook
└── translations/
    └── fr.ts             # French translations
```

## Adding a New Language

### 1. Add Language to Enum

Edit `src/i18n/types.ts`:

```typescript
export enum Language {
  French = 'fr',
  English = 'en',  // Add this
}
```

### 2. Create Translation File

Create `src/i18n/translations/en.ts`:

```typescript
export default {
  common: {
    cancel: 'Cancel',
    confirm: 'Confirm',
    // ... copy structure from fr.ts
  },
  auth: {
    login: 'Login',
    // ...
  },
  // ...
} as const;
```

### 3. Register in i18n Config

Edit `src/i18n/i18n.ts`:

```typescript
import en from './translations/en';
import fr from './translations/fr';

const i18n = new I18n({
  fr,
  en,  // Add this
});
```

### 4. Add to Language Selector

Edit `src/components/ui/LanguageSelector.tsx`:

```typescript
const AVAILABLE_LANGUAGES = [
  { code: Language.French, name: 'Français', flag: '🇫🇷' },
  { code: Language.English, name: 'English', flag: '🇬🇧' },  // Add this
] as const;
```

### 5. Update Translation Keys

Add the language name to `src/i18n/translations/fr.ts` and other files:

```typescript
languages: {
  fr: 'Français',
  en: 'Anglais',  // Add this
}
```

## Translation Keys

Keys are organized by feature domain:

- `common.*` - Shared UI elements
- `auth.*` - Authentication & user management
- `profile.*` - Profile screen
- `journey.*` - Journey tracking
- `spots.*` - Spot management
- `map.*` - Map interface
- `languages.*` - Language names

## Persistence

Language preference is automatically saved to secure storage and restored on app restart.

## Best Practices

1. Always use translation keys, never hardcoded strings
2. Keep keys organized by feature domain
3. Use interpolation for dynamic values: `t('key', { value: 'data' })`
4. Test all languages before release
5. Keep translation files in sync (all should have same keys)
